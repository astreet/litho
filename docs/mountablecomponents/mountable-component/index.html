Creating a Mountable Component

Mountable Components can be used to render views or drawables on the screen.

Comparison with KComponentsâ€‹

There are several similarities between Mountable Components and KComponents:

  • They both consist of a render function that can use the component's props and hooks.
  • As with KComponents, mountable components are subject to rules for their use.
  • Just like KComponents, mountable components can be configured using common props and state.

Creating a Mountable Componentâ€‹

The following example shows a Mountable Component that will return a 'mountable' that creates an 'ImageView' to be rendered on the screen.

class SimpleImageViewComponent(private val style: Style? = null) : MountableComponent() {

  override fun MountableComponentScope.render(): MountableRenderResult {
    return MountableRenderResult(SimpleImageViewMountable(), style)
  }
}

A Mountable represents a reusable unit responsible for hosting the logic to create, measure, and bind the content that the Mountable Component will render. For more information, see the Mountable page.

As can be seen from the above example, the render() function returns:

  • A MountableRenderResult object that contains a 'Mountable' to be rendered
  • A style object that is applied to the mountable component.
note

The render() method can be invoked from multiple threads. You should not assume it can only be called from the main thread. The implementation of render should not cause side effects; essentially, it acts like a pure function.
